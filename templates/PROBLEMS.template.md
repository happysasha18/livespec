# <project> — Problem ledger (`.live-spec/PROBLEMS.md`; SPEC E-24, INV-23)

The workshop's dynamic list: operational noise — a flaky test harness, a missing dependency, an
environment error, a tool misbehaving. Never the product's own defect: that is a bug and takes the bug
lane. The walk, the moment noise fires: grep this file for the signature. Not listed → add one WATCHED
line and keep working — the write replaces the silent retry. Listed → the SECOND occurrence gets an
owner THAT MOMENT: a queue row (OWNED, row #) or the human's dated AGREED NON-PROBLEM (his word alone).
A third unowned recurrence is a defect of the METHOD — one wish file to the pack's inbox, citing the
signature and its dates. A recurrence on an owned entry appends its date and changes nothing else. The
landing that closes an OWNED entry's row flips it to SOLVED. At the milestone compaction, SOLVED and
agreed entries move to the dated ARCHIVED tail below, and duplicate signatures merge.
