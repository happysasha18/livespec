## Ratio

sample.md / source total = 12,000 / 17,297 = **0.694**.

The new genre reproduces every condition of the subsection and every cited index row in about 69% of the source bytes. The saving comes almost entirely from the index rows: the prose subsection is close to its EARS form in size, but the 11,019 bytes of dense index rows collapse into short one-trigger-one-response lines. The rows are where the current format hides its cost, and this genre spends that cost as explicit, addressable criteria instead of one long run-on sentence per code.

## Projection to the whole spec

Assumption: the whole 783,678-byte spec is the same material as this subsection — human-facing rules each backed by a dense index row — so the 0.694 ratio holds uniformly across it. This is a rough assumption: the real spec also holds tables, journal history, and machinery sections that this genre would treat differently (history is dropped entirely, which would push the ratio lower, while wide tables recast as criteria could push it higher).

Under that flat 0.694 assumption, the whole spec in this genre would be about **543,685 bytes** (≈ 0.54 MB), a reduction of roughly 240,000 bytes.

If the history-drop rule alone were counted (this genre omits dates, provenance, and prover-finding notes that fill a large share of the index rows), the real figure would land below this projection.
