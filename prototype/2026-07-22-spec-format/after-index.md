| Anchor | Rule |
|---|---|
| INV-150 | Every declared cross-cutting law names the one net that enforces it — a mechanical gate, the prover's judgment station, or the design review's recommendation — and declaring a property is what promotes it from a soft watch to a blocking net. related: INV-101 |
| INV-151 | A request enters the pipeline at the highest document whose sentences must change to satisfy it, the set of entry doors being closed on purpose, so a request that matches no door becomes one plain question to the human rather than a guessed route. related: INV-4 |
| INV-153 | Every arriving item — a request, a property to enforce, a deferred row, or a message leaving one project for another — routes to the one home whose declared sentence governs it, and an item that fits no home is itself the finding rather than something forced into the wrong place. related: INV-151, INV-150, INV-152, INV-189, INV-191 |
| INV-104 | A fix that touches a spec-backed literal — a version string, a pinned count, a named vocabulary, a promised wording — owes its documentation update and its red-first test in the same session as the fix, whatever the size of the diff. |
| INV-128 | Every request gets a three-source impact read at intake — the spec for what behaviour changes, the architecture for which module owns it, the code for what actually gets touched — producing one named footprint of presentation-only, single-module, or cross-cutting that sizes how far the route reaches. related: T-12, T-16, INV-43, INV-108, INV-37, INV-121 |
| INV-144 | When the product and the spec diverge the spec is the definition of correct, so a wrong product is fixed to it and a spec silent where the product is right is completed with a pinned test, while changing a spec confirmed to be the error takes the human's word. related: INV-9 |
| INV-129 | Every deferred row's revisit trigger is re-scanned at each queue-take instead of only at milestones, and a fired trigger returns its row to the runnable head, so a deferred wish never waits on a trigger nobody reads. related: T-8, INV-49 |
| INV-130 | A withdrawn decision converges after two withdrawals of the same question, at which point the session takes the recommended option, surfaces it as a default in the landing report, and closes it so it is never re-asked. related: INV-59, INV-9, INV-31 |
| INV-131 | A mid-work re-door that creates a new surface re-runs the independence checks between the parallel lanes, and a fresh collision pulls the re-doored lane back to serial with the departures board saying so, so the board never asserts a stale independence after the ground moved. related: INV-16 |
| INV-133 | A critical mark on a non-bug wish sends it to the head of the queue while the rolling lane keeps running, and the capture echo says that bound back at intake so the human can re-door it a bug if he meant a live break that must stop the work now. related: T-11, INV-27, INV-9 |
| INV-132 | Every H3 heading in the spec carries either its feature tag, marking a person-facing scenario the coverage table maps, or an explicit not-a-scenario marker for machinery and reference sections, so a heading carrying neither goes red and a forgotten scenario tag can no longer ship uncovered. related: INV-73, E-29 |
| INV-134 | A landed feature-or-refactor row carries the footprint note its intake read named, and a suite check reddens any landed row that omits it, putting a mechanical floor under the footprint read. related: INV-128, INV-159 |
