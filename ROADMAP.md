# live-spec Roadmap (dated version: 2026-07-05 — updated at every edit; SPEC M-3)

The wish queue. Intake is continuous — a wish lands here the moment it is spoken. Execution is serial:
the current landing finishes before the next starts. Class = size (bug / small / surface / large — one
vocabulary with the spec) plus a priority mark when it isn't normal: a critical bug lands before
everything; a quick win may bubble up between landings, the jump marked in its row (SPEC T-11, INV-12).
A row also names its DOOR — feature · bug · refactor · docs-only · skip — said at intake, before any
code; feature tripwires outrank a casual label and queue-cutting stays with the bug door (SPEC T-12, INV-16).
