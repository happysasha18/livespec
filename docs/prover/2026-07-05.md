F2 — "Rows are never deleted" will collide with "nothing grows unboundedly"

> "rows are never deleted — only closed with a named exit" — Throwing a wish [INV-1]
> "doc COMPACTION (pruning: redundancy removed from spec/matrix/queue/skills — nothing grows
> unboundedly)" — The rhythm [M-1]

The queue is at 28 rows after three weeks; at a year it is hundreds of closed rows, and the milestone
compaction rule names the queue as a pruning target while INV-1 forbids removing rows. An agent obeying
both either skips queue compaction forever or violates INV-1.

State the reconciliation: closed rows MOVE at milestones to a dated archive (`ROADMAP.archive.md` or a
closed-rows section), never edited, never lost — "never deleted" means never lost from history, mirroring
the attic principle. DISPOSITION: queued — ROADMAP row 30.

`should-clarify · internal-conflict (consistency)`

F3 — "On any version change" has no defined place a version lives

> "On any version change (livespec or any installed skill), the agent RE-READS the changed SKILL.md" —
> Adoption step 7 [A-7]

Neither the package nor any of the four skills carries a version number anywhere on disk, so the A-7
journal note "naming old → new" cannot be written; only the mtime half of the re-stat is executable today.

Either add a canonical version home (a `VERSION` file for the package; a `version:` frontmatter line per
SKILL.md) or reword A-7 to mtime-based detection until versions exist. Recommend the version homes — the
re-read trigger should survive a file touch that changes nothing. DISPOSITION: queued — ROADMAP row 31.

`should-clarify · unenforceable-promise (discharge)`

F4 — Two bugs at once: the preemption rule doesn't order them

> "the bug takes the lane, and the parked wish resumes as the immediate next landing" — When a bug cuts
> the line [T-9]

A second bug arriving while the first holds the lane meets two rules that both claim the next slot: "a
bug may interrupt" and "the parked wish resumes as the immediate next landing." The agent picks silently
— exactly the micro-decision class INV-5 forbids.

One sentence in T-9's section: bugs take the lane in arrival order; the parked wish resumes only when no
bug waits. DISPOSITION: FOLDED with this record (no new anchor; anchor set unchanged, guard checked).

`should-clarify · undefined-path (transitions)`

F5 — The global-default profile has no stated home

> "at `.livespec/profile.md` in the host (global default as fallback)" — Who decides what [E-8]

Communicator is told to read the profile before every exchange; in a host with no profile yet it falls
back to a global default whose path exists nowhere, so the fallback silently becomes "no profile" —
mode/trust/language revert to the agent's guess, which INV-9 exists to prevent.

Name the global default's path when the profile mechanism next lands (candidate: alongside the global
CLAUDE.md); until then E-8 should say the fallback is ⟨DECIDE⟩-pending. DISPOSITION: queued — ROADMAP
row 32 (small, lands with the profile work).
