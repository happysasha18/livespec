### F1 — the fold

> "The universal tier — the scissors ban, the negation-opener rule, and the machine-jargon rule — states the plainness every live-spec document holds whatever its register, so it binds every host's gate." — PRODUCT_SPEC.md:772 (Rhythm / "The style lint has two tiers")

The spec claims two tiers into which "the style gate's rules divide by whom they bind." `provenance-narrative`
binds every host in every tier (it is a hard error even in default mode) but appears in no tier of the
spec's taxonomy. The implementation is internally correct and documents the exception; the gap is the
spec prose presenting a closed two-tier partition that the rule set does not honour.

Fix: add one half-sentence to the clause naming `provenance-narrative` as a structural rule that binds
every host unconditionally, outside the register split (the linter comment at `spec-style-lint.py:242-243`
already states the reason — a birth-story in a normative body has no register reading). This makes the
tier taxonomy account for all eight rules. Behaviour, tests, and `--gate` back-compat are untouched.
