## Adversarial read (set on breaking it)

A fresh seat sampled fifty-seven descriptions across every kind — INV, E, T, M, A, ACT, B, C, D, S —
including the dense codes (INV-128, T-18, INV-236, INV-183, INV-101, C-1) and the class laws, scanned all
327 rows mechanically for structure and register, and empirically mutated scratch copies to test the gate.

- **Structure — clean.** Every one of the 327 data rows carries exactly four cells; the header and
  separator are four-column; no duplicate anchors, no missing or extra codes, no pipe character leaked
  into a description (a leak would have split a row to five cells — none did). The range row T-1..T-7 is
  well-formed with a single-sentence description. Every description is one sentence ending in one period,
  with no markdown or anchor leak.
- **Register — clean.** No contrast frames stating a thing against a denied neighbour. The em-dashes
  introduce appositive representative-member lists the accepted form permits. No coined mechanism-metaphor,
  calque, or verbatim quote; the register lint and the spec-style lint both report zero.
- **Gate honesty — verified by mutation.** Against the real armed config, an emptied cell, a deleted cell
  (short row), a placeholder, a single-word field, and a pipe-smuggled five-cell row all exit 1 and name
  the code; a smuggled pipe does not slide the Section text into the description slot, because the slot is
  pinned by header index. The two dormancy tests were genuinely rewritten to assert the armed state and the
  real tree passing, not gamed. 17/17 description-field tests pass.
- **Accuracy — strong.** Fifty-five of fifty-seven sampled descriptions are faithful to their declaration
  prose. The pass blocked on one real defect and folded two nits:
  - the byte-bound reason recorded the growth as "~36 KB" when it is ~91 KB (642 KB to 736 KB) — a false
    figure in the audit trail of a permanent cap bump on a major release; corrected in `doc-bounds.json`
    and the MIGRATION chapter;
  - INV-189 had named two of its closed set of three message births; the third (a concern no agent's zone
    owns, carried to the pack) was restored;
  - INV-92 said "anchor set" where the catch-up walk records an "anchor multiset"; corrected.
