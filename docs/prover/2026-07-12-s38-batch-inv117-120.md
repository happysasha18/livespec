## The two concurrency fixes, against the rest of the model

**INV-117 closes F1 with no new seam.** The old wording broke the pen tie-break onto a session token that
only existed after an inbox collision (mint point at line 1596 in the pre-fix text); a session that never
raced an inbox slug carried no token, so the tie-break was undefined — the mutual-back-off or double-write
F1 named. The fix mints one identity before a session's first act, so every session carries one by the
time it can claim a lane. Distinctness holds in every case: a harness session id is unique by
construction; a minted identity joins the start time, the worktree path, and a nonce, and distinct
worktrees already differ under INV-105 while the nonce separates two sessions sharing a worktree and a
start second. A lexicographic order over distinct identity strings is total, so exactly one of any pair
sorts lower. The tie-break stays symmetric: the row→in-work flip records the claiming session's identity,
both concurrent flips commit to the shared tree as ancestryless siblings, so each peer reads both
identities and computes the same lower-holds verdict from either side — mutual back-off cannot happen and
no two sessions take the pen. The channel is the existing fence machinery (INV-2, INV-11); the inbox
source-mark keeps its shape and only sources its token from the identity now. Nothing new crosses between
two nodes.

**The INV-76 amendment closes F2 with no new seam.** F2's false-death case was a compute-bound worker
that legitimately writes no product file for minutes and may miss the probe window, reading dead on both
original signals though it was live. The amendment adds a third signal — the worker touches its checkpoint
file every ~60 s as a bare heartbeat (worker-contract line 1410, under ACT-3, itself base-rulebook) — and
requires all three quiet together for a dead verdict: a still write-set, a stale heartbeat, and an
unanswered probe. The intervals compose: a live worker touches twice inside any ~2 min staleness window,
comfortable margin against one missed touch, so the compute-bound run reads alive while a truly gone
worker's heartbeat goes stale. It composes with INV-95 too — a worker dies with a sleeping machine, its
heartbeat stops, and after resume the stale heartbeat correctly reports the death INV-95 already declares.
The heartbeat lands on the worker checkpoint file the resume protocol already reads; no new node, no new
crossing.

**F1 and F2 do not collide with each other.** INV-117's identity lives in the session checkpoint;
INV-76's heartbeat lives on a worker's own checkpoint file. Both sit under `.live-spec/checkpoints/` but
key on different owners (per-session versus per-worker-id), so they never share a file, and one is static
while the other ticks — no timing interaction.
