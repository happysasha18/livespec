## Findings

F1 — The lane token didn't actually serialize two parallel sessions
> "the single in-work row IS the lane token, so any assigned session … takes a wish only after seeing no
> other row in-work" — Throwing a wish, INV-2
Claiming the lane was a bare read: two sessions could both see the queue head free, both flip it in-work
in their own trees, and both land — a double landing on the flagship repo, discovered only at push. The
INV-11 fence only trips on committed HEAD movement, and nothing made the claim itself a commit.
Fold: claiming the lane IS an atomic committed act — commit the row→in-work flip first, re-check the
token under the fence immediately before drafting the spec-delta, and back off if the re-check reveals
another committed in-work row. Serialization now pins to committed history, never a bare read.
`must-fix · boundary-issue (concurrency)` → **FOLDED** (INV-2 clause)

F2 — A deferred wish could be archived before its revisit trigger fired — a lost wish
> "at a milestone, closed rows MOVE to a dated queue archive — archived, never edited" — INV-1
Deferred is non-terminal (it carries a revisit trigger), but the milestone archive swept every closed
row; a trigger firing after the sweep pointed into a frozen archive nobody re-scans — the flagship
"no wish is ever lost" violated by its own archive clause.
Fold: only TERMINAL exits (landed · declined · superseded) archive at a milestone; a deferred row stays
in the active queue until its trigger fires or it re-resolves to a terminal exit.
`must-fix · internal-conflict (consistency)` → **FOLDED** (INV-1 clause)

F3 — An inbox wish's arrival time was undefined against spoken wishes
> "critical bugs head the waiting line (among themselves by arrival)" — T-9, with T-11's tie-break
File date vs harvest moment gave two critical bugs from different doors a nondeterministic order.
Fold: an inbox wish's arrival IS its harvest moment — when it first becomes a row the ordering rules can
see; filename order breaks ties only within one sweep.
`should-clarify · undefined-path (ambiguity)` → **FOLDED** (T-11 sentence)

F4 — The harvest idempotency parenthetical guarded an unreachable state
> "which re-harvests idempotently (a file already represented by a row is removed, never re-added)" — T-10
Atomic one-commit harvest + the -2/-3 naming law make "file present AND row exists" unreachable; an
implementer would build a recovery path that can never fire.
Fold: reworded to the single reachable recovery — an interrupted harvest (nothing committed) leaves the
file untouched; the next sweep harvests it exactly once; a committed harvest leaves no file behind.
`worth-considering · over-specific (atomicity)` → **FOLDED** (T-10 sentence)
