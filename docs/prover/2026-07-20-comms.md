## Findings

### B1 — BLOCKER — INV-240 (line 1454) — the deferral fold is incomplete and self-contradicting; the same-window overwrite reopens N2

The N2 resolution the author chose was to DEFER the overwrite (records the re-question, rewrites on
the owner's next penned run, takes the pen [INV-39/198], rides as a named delta to the restructure
gate [INV-111]). The current text binds every one of those protections to the CROSS-window path
only — "the owning agent reformulates and overwrites the description on its own next penned run.
The overwrite waits for that penned run ... that penned write takes the description home document's
pen before it lands [INV-39, INV-198] and rides as a named intended delta to the
restructure-identity merge gate [INV-111]." The word "that" ties the pen and the named delta to the
cross-window "next penned run."

The same-window path keeps the reactive language and names neither protection: "When the confusion
lands in the owning window, the owning agent reformulates ... and overwrites it directly in its one
home in place [base rule 4]." No pen, no named delta, "directly ... in place." And the clause's own
bolded lede states the general law as immediate — "A description that leaves a reader asking what a
term means is rewritten **at that moment, in place**." — which the Formal-index row 2371 repeats
verbatim ("rewritten at that moment in place").

So the delta asserts two incompatible timings at once: the lede + the same-window sentence + the
index row say "at that moment / directly in place" (reactive mid-turn), while the deferral body says
it "waits for that penned run" and "stays clear of firing reactively in the middle of another turn."

What it breaks: the Formal index is a section of the penned shared document (PRODUCT_SPEC.md). A
same-window overwrite fired "at that moment" is exactly the unfenced reactive write N2 named — it
can fire while another lane holds the pen (the pen is per-document, INV-198; the owning agent's live
session does not necessarily hold it), and it changes description tokens the restructure-identity
gate [INV-111] never authored, the literal-drift class this pack has been bitten by. The cross-window
path is now protected and the same-window path is not, though both write the same penned document.

Fix (either, applied to BOTH paths): (a) make the deferral general — the owning agent records the
re-question and overwrites on its next penned run whether the question landed in its own window or
arrived by earned message, and every such write takes the pen and rides as a named delta; rewrite the
lede so it no longer says "at that moment." Or (b) if a same-window overwrite is meant to fire during
the owning agent's own live penned run, say so and still bind the pen [INV-198] and the named delta
[INV-111] to it — the fence and the restructure gate do not care which window originated the write.
As written, the pen and the named delta are named once, for the cross-window case alone.

### B2 — RECOMMENDATION — INV-240 (line 1454) — "first-birth earned message" mislabels the fault birth the same sentence describes

The cross-window route reads: "that window carries the confusion to the owning agent as a
**first-birth** earned message [INV-189] — the reader's confusion is a **lived fault** the window
carries with the evidence in hand." Under INV-189 the births are enumerated blocked-first (line
1540: "The sender is blocked by the receiver's zone as it stands"), fault-second (line 1541: "The
sender lived a fault in the receiver's zone and carries the evidence it lived"). A "lived fault ...
with the evidence" is the FAULT birth (the second), not the first.

The category itself is NOT forced — a confusing description genuinely is a lived fault in the owning
agent's zone, and the non-owning window genuinely lived it (its reader asked), evidence in hand. The
route composes. Only the ordinal word is wrong. But it matters: the two births "owe different things"
(line 1544 — the blocked message owes work standing still, the fault message owes the evidence), and
the receiving gate "reads the named blocked work" (INV-189 index). Tagged "first-birth" (blocked), a
confusion message would be checked for blocked work standing still, which it does not have. Name it
the fault birth to keep the gate keyed on the evidence it actually carries.
