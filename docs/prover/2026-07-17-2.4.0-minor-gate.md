## Phase 3 — Property analysis

R1 — The row-366 hygiene net is a best-effort static scan; the spec's "the reach stays whole" overstates what it enforces

> "a test that reads an infra directory by walk or glob rather than naming its files' basenames is pinned into an always-scoped list ... so every such test rides every scoped run and the reach stays whole [ROADMAP 366]." — PRODUCT_SPEC.md, INV-45 middle-road paragraph (line 1683)

The net (`_enumerating_infra_tests`) flags an enumeration only when the infra directory name appears as a **literal token inside the enumeration call's argument text**. A test that binds the directory to a variable first — `d = "guardrails"; glob.glob(d + "/*")`, or `os.walk(some_infra_dir)` — presents call args of `(d + "/*")` / `(some_infra_dir)` with no infra literal, so the scan does not flag it, does not force it to pin, and it can still escape a scoped run. The spec's universal "every such test ... the reach stays whole" reads as a guarantee, where the net is a best-effort approximation of it. The production impact is nil today (the audit that spawned row 366 found zero live instances, and the evasion pattern is contrived), so this queues rather than blocks. Fold, if taken: soften the clause to name the net's scan basis ("a test that names an infra directory as a literal in its enumeration is pinned ..."), so the claim matches the mechanism; or leave it and record the residual static-scan bound as a decided sentence.
