**2. The gate that reds serial independent work — DECIDED as a DISCIPLINE, with the reasoning, and NOT shipped as a hollow gate.** This is the hard part, and the honest verdict is that it cannot be a sound mechanical gate. Three reasons, each concrete:

  - **Independence is a senior judgment, not a diff.** Deciding that two rows were independent and *should* have rolled in parallel IS the independence graph [INV-49] — an edge wherever two rows share a surface, a spec section, a skill file, or a doc region, computed from a prediction of each row's write-set. That prediction is a read, not a mechanical fact a gate can recompute.
  - **The evidence a right run would leave is destroyed by design.** A lane branch is torn down at its landing [T-23, INV-199]. So at any push-gate moment `git branch --list 'lane/*'` shows the currently-open lanes, never whether lanes *were* used for already-landed rows. The reflog is local and gc-prunable, so no durable push-time signal survives to key a gate on.
  - **The one mechanical signal on offer reds every lawful serial movement.** The candidate "≥2 rows landed with no `lane/*` branch cut → red" fires on a dependency chain, a shared surface, and a pair of tiny rows (INV-49: "tiny rows ride serial"), all of which SHOULD be serial. Adding a "unless a reason is recorded" escape makes the gate quiet only when a reason exists — which reduces it to a prose scan that still cannot judge whether the recorded reason is *correct*, since that judgment is the independence read again.

  So the check stays a **discipline** the session holds: when the graph shows two or more independent runnable rows with lanes free under the cap, the session performs the act, and going single-file then is a recorded "serial by the graph" board line naming which standing reason holds. This is row 420's own audit rule applied to itself — a taste/judgment call is never a gate — and the meta-gate w would red a hollow gate that cannot truly fire, so shipping one would be caught anyway. No new gate letter is minted, no entry is added to `gate-red-proofs.json`, and the meta-gate's coverage is unchanged. The one mechanical guard the act genuinely carries — the cap refusal — is deterministic (a count of open `lane/*` branches against the profile cap) and IS built and tested by deed.
