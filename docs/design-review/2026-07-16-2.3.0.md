## The one question (finding 1), both objects in hand

`INV-157` (:666) adds a **launch frame probe**: before any suite is trusted, one throwaway page awaits a single
compositor frame within ~2 s, and a browser that produces no frame fails loudly under the probe's own name
rather than bleeding false reds through the suite's paint-waiting rows. The **same incident** that bore this
delta named a **second, independent** environmental fault (ROADMAP 364): Chrome for Testing 151 *renders frames
yet stalls loading any 127.0.0.1 page*. The clause protects the fallback path against the first fault with a
probe on every binary; the second fault is protected only by binary resolution preferring the shell — and
resolution **falls back** to Chrome for Testing then a system Chrome, the exact build family that carries both
faults. On that fallback the frame probe passes and a localhost-stall bleeds the same confusing half-red the
frame probe was built to end.

**Should the launch-probe kind carry a second member — a page-load probe (a throwaway navigation to a served
127.0.0.1 page that must load within a bound, failing loudly under its own name) — so the fallback binary is
guarded against both named faults; or is the page-load stall deliberately left to binary resolution alone (the
shell is the pick, a fallback that stalls is an accepted degraded mode), recorded as a decided sentence so the
ask never re-fires?**

Recommended default: **add the page-load probe as a lighter second member**, or fold it into the frame probe by
pointing the one throwaway page at a served 127.0.0.1 URL and awaiting both its load and its first frame. Ground:
the incident named two independent faults; resolution's fallback lands exactly on the build family that carries
both; the frame probe already proves the pack will pay for a launch-time throwaway-page check, and a browser
that renders frames but will not serve localhost bleeds precisely the confusing half-red the probe exists to
end. Keep it lighter than a per-page assertion — one bounded navigation. If the human reads resolution as
sufficient (the shell is the default, a fallback stall is a rare accepted degradation), the answer is
"different, by intent — resolution owns the page-load fault," recorded as a decided sentence [INV-59] so the ask
never re-fires. Either answer lands a declaration; the class sentence lands only on the human's word [INV-142],
and the lane is not held.
