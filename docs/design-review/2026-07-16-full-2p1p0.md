## The one question (finding 1), both objects in hand

**INV-27** (:124) promises every captured wish its echo, and routes a silent arrival's echo to the
next status report. **INV-146/147** (:1717–1719) give the stranger arm a recorded note only on the
no-wish verdict; a captured stranger wish leaves its source Issue open and silent — the stranger
cannot read the host's status reports, so the echo law's channel does not reach the one depositor
class built for outsiders.

**Should the stranger arm carry the echo back to its source — the capture echo (heard · door ·
name · row) posted as a comment on the source Issue at harvest, and the Issue closed when the row
reaches a terminal exit — or is the stranger deliberately unanswered beyond the no-wish note, the
door being intake-only by intent?**

Recommended default: **post the echo and close on terminal exit.** The write rides the comment
grant the monitor already holds [INV-146, INV-148], so it costs no new machinery; the echo law's
own ground ("this immediate echo lets the user know the request was received") applies with the
most force to the depositor with no other channel; and a closed Issue is the same convergence an
answered question already has [INV-59]. The class sentence lands only on the human's word
[INV-142]; the lane is not held.
