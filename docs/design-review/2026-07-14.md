# Design review — live-spec, 2026-07-14 (the first M-1 design-review record)

**Design-reviewer skill version this pass ran under:** v1.0.0 (`skills/design-reviewer/SKILL.md`).

**What this is.** The design review is the pass that runs right after the prover on a re-proven spec and
judges the design itself — do same-kind things behave alike, and what groupings did the text never
declare [INV-141]. Chunk 3 of the cleanup movement wired it into the M-1 milestone gate, so the 1.4.0
milestone is its first firing. This is that record.

## The pass stood down — by its own kind-scaling rule

The design review builds its inventory from **every element a person acts on that a spec sentence names**
— a photo, a caption, a control, a slot — and proposes the same-kind groupings the surface list never
declared. Its whole subject is acted-on product elements one level below the page-level surface registry.

live-spec is a **skill pack**: a rulebook, nine working skills, guardrail scripts, and templates. It has
no surfaces a person acts on — no photo, no caption, no control, no slot. Its "elements" are invariants,
rules, and lenses, which are the design review's *subject matter*, not the acted-on UI elements it groups.
There is no element inventory to build below the registry, so there is no same-kind grouping to propose
and no behaviour parity to check.

INV-141 states this case directly: *on a kind with no elements a person acts on, the design review stands
down by name in the record rather than running vacuously and reading as coverage it did not give* [INV-141,
INV-22]. This record is that named stand-down.

**Verdict:** the M-1 design review on live-spec's re-proven spec **STANDS DOWN** — live-spec is a skill
pack with no acted-on elements, so the pass has no inventory to run and records its stand-down here rather
than running vacuously [INV-141]. No findings. No asks held. The same-kind design questions this pass
exists to catch belong to the *products live-spec serves* (a visual product, a gallery, a site), where the
adopting project runs its own design review over its own acted-on surfaces.
