## The same-kind group the text never declared

**The suite-honesty / test-infrastructure invariant family** — INV-77, INV-78, INV-79, INV-80, INV-100,
INV-102, INV-155, INV-157, INV-158. Shared role, one plain sentence: *a law that closes a way the suite
could pass green while the fact it claims is false, or leaves the machine worse than it found it.* No clause
names them as one class or states their parity; INV-80 uses "one class" only for its own three internal legs,
not for the family. Under that role the members split into three sub-roles, and the parity that matters runs
**within** the sub-roles:

- **Assertion-correctness** (a green assertion that proves nothing true): INV-77 device boundary, INV-78
  geometry drift, INV-79 engine-on-donor-data, INV-102 mirror expected value.
- **Plumbing / harness honesty** (the harness lies about its verdict or misbehaves on its own run): INV-80
  plumbing must not lie, INV-155 flaky masked, INV-157 harness sound/orphan/timeout.
- **Hygiene** (the run leaves the machine worse): INV-100 cleanup + temp home, INV-157 mute + reap orphans.

## Parity check — does each member name the net that catches its violation?

The un-declared but plainly-carried property across the family: each member names a **net that fails a run on
a regression**, not just the fix.

| member | its net that catches a violation |
|---|---|
| INV-100 (hygiene) | mechanical: suite-prefixed temp artifacts + session-scoped before/after diff of the temp home fails the run on a surviving file |
| INV-80 (plumbing) | mechanical, per leg: skip-helper imports at module load; re-export completeness test; gate reads the log's tail line not the wrapper exit |
| INV-155 (flaky) | two nets: mechanical grep for a retry/rerun plugin + verify-walk discipline; quarantine in the pinned skip-set with a dated reason and owning row |
| INV-102 (mirror) | the prover/verify read — three legal expected-value sources named |
| INV-77/78/79 | the assertion shape *is* the net — relative+wide+long geometry, engine-shaped fixtures, the real-device walk row the suite can never green |
| **INV-157** (mute/reap/deadline) | **names the fix (mute flag, reap the process group, bounded per-command deadline) and the not-removable → ledger seam [INV-23] — but names no net that fails a run when the harness leaves an orphan alive or plays sound** |

The gap: INV-157 explicitly frames its mute+reap obligations as INV-100-class hygiene ("launching muted and
reaping the children are test hygiene, a run leaving the machine as it found it [INV-100]"), but INV-100's own
mechanical net is a before/after diff of the **temp home** — files. An orphan process is not a temp-home file;
a played sound is not a file. So INV-157 inherits INV-100's *framing* but INV-100's *net* does not reach
INV-157's two new obligations, and INV-157 names no substitute. Its plumbing-sibling INV-80 gives each of its
three legs a concrete mechanical check; INV-155 names two nets. INV-157 alone names only the fix.

## The one question (finding 1), both objects in hand

INV-157 says its harness obligations — starting the browser muted, reaping the whole process group on
teardown — *are* test hygiene, "a run leaving the machine as it found it [INV-100]." INV-100 holds its own
hygiene obligation with a mechanical net: a session-scoped before/after diff of the temp home fails the run on
a surviving file. But that diff sees files, not orphan processes or audio, so it does not catch an INV-157
violation. **Should INV-157's mute + reap obligations carry a mechanical check that fails a run — a before/
after process-group diff after teardown, an assertion the launch flag was passed — the way its declared
sibling INV-100 does and the way INV-80 gives each plumbing leg its own check, or is INV-157 deliberately
verify-walk-only, root-fix-here with no run-failing net?** Recommended default: give INV-157 a mechanical net
of its own (a post-teardown orphan-count check on the harness's process group, red on a survivor; the launch
asserts its mute flag is set), so a harness that regresses into playing sound or leaking orphans reds a run
rather than passing green — the same mechanical floor INV-100 and INV-80 already stand on.

## Strongest divergence (two sentences)

INV-157 declares its mute-and-reap obligations to be INV-100-class test hygiene, but INV-100's only mechanical
net is a temp-home file diff, which cannot see an orphan process or a played sound, and INV-157 names no
substitute — so where INV-100 and INV-80 each fail a run on their own violation, a harness that regresses into
leaking orphans or making noise passes INV-157 green. The illusion the whole family exists to close reappears
inside its newest member, on the exact obligation it borrowed the sibling's name for.
