## Findings

**1. BLOCKS-PUSH — the launch probe does not fail "under its own name" on the interpreter this machine actually runs.**
Evidence: `templates/headless_harness.py:775` and `:799` catch `except TimeoutError`; but the deadline machinery's common expiry path is `sock.recv()` raising `socket.timeout` after `settimeout(remaining)` (`templates/headless_harness.py:505–509`). On Python < 3.10 `socket.timeout` is an `OSError` subclass, **not** `TimeoutError` — verified by deed on this machine's `python3` (3.9.6): a timed-out recv raised `timeout`, `isinstance(e, TimeoutError) == False`. So a frame-dead or loopback-stalled browser escapes `Browser.__init__` as a raw `socket.timeout("timed out")`, never `FrameProbeFailed`, never naming the leg or the shell install command — the exact behaviour INV-157 (PRODUCT_SPEC.md:666), M-343 (TEST_MATRIX.md:519 — "the message naming which leg failed"), and ROADMAP row 364's Done-when claim. The only path that raises builtin `TimeoutError` is a deadline already spent *before* a recv begins (`:506–507`) — a race, not the stall case. This is exactly the string-grep trap the brief asked me to name: `test_probe_failure_names_itself` (tests/test_harness_template.py:314–325) greps `FrameProbeFailed`/message strings in stripped code and stays green while the wrong exception type ships. The 364 smoke-run ("both legs pass") only exercised the green path. Fix is one line per leg — `except (TimeoutError, socket.timeout)` — plus, ideally, a by-deed red on the exception type. This is a delta divergence from the row's own law, so it blocks by the pack's own equivalence rule. (On Python ≥ 3.10 the code is correct as written.)

**2. QUEUE-ROW — the scoped road's by-name discovery is blind to enumerating tests; INV-40's flat claim holds only contingently.**
PRODUCT_SPEC.md:1682 asserts "the scoped run is still every check the diff can reach [INV-40]", but `check-push-reach.sh:123–138` finds owning tests only by basename grep. A test that reads an infra dir by glob/walk *without naming basenames* would read the changed file yet never be picked. I swept for live instances: the enumerating tests (`test_delegation_trigger_no_size.py:41`, `test_prose_gate.py:301`, `test_traceability.py:683` etc.) walk `skills/` and `docs/` only — **no current test enumerates guardrails/, templates/, scripts/, hooks/, or adopt/ wholesale**, so there is no live violation today. But nothing prevents the next such test from silently falling outside every scoped run. Deserves its own row: either a suite-hygiene net (an enumerating test over an infra dir must be pinned into the scoped set) or a decided sentence softening the INV-40 claim.

**3. NOTE — the meta-reach classifier drops a renamed origin.**
`_push_diff_files` (tests/test_guardrails.py:82–83) keeps only the new side of a porcelain rename, and `git diff --name-only` with rename detection likewise lists the new path only — so gate machinery renamed *out* of `GATE_MACHINERY_PREFIXES` (e.g. `guardrails/x.sh` → `scripts/x.sh`) could skip the meta-test. Marginal in practice: such a move almost always also edits `guardrails/pre-push`, which is in class.

**4. NOTE — a probe failure leaves the just-launched Chrome alive until interpreter exit.**
`FrameProbeFailed` raises out of `Browser.__init__` (templates/headless_harness.py:631–632) after `Popen` (:603); `close()` never runs on that path. Reaping falls to the atexit hook (the instance is in `_LIVE_BROWSERS`, :570) and a sibling launch's sweep correctly skips the live owner — so nothing leaks past process exit, but a frame-dead Chrome idles for the remainder of a long pytest run. A `try/except: self.close(); raise` around the probe call would be cleaner.

**5. NOTE — the fallback frame leg is bounded by 60 s, not "~2 s".**
On the no-loopback fallback road (templates/headless_harness.py:787–791) the `data:` `Page.navigate` passes no `timeout`, so `CMD_TIMEOUT = 60` governs it, against the spec's "each leg under its own bounded ~2 s" (PRODUCT_SPEC.md:666). Rare road, safe direction, but a stated-bound miss.

**6. NOTE — a diff that deletes a test file scopes to the deleted file.**
`check-push-reach.sh:113–117` adds a changed `tests/test_*.py` to the scoped set as its own owner; a *deleted* test file then makes gate b's pytest red on collection — a false red, conservative direction only.
