# Wish: an architecture REWORK owes a rework of ARCHITECTURE.md itself — state it as a law

**The item.** When the structure of a product is deliberately redesigned (layers restacked, ownership
of a surface moved, nodes merged or split), the pack should require the architecture DOCUMENT to be
reworked with it — restructured to the new shape, re-proven with the architecture lens, in the same
movement as the redesign. Today the pack states only two narrower things: build-pipeline step 3 says
"write or update ARCHITECTURE.md from the proven spec" for a change riding the pipeline, and the
refactor line says "if the refactor moves node boundaries, ARCHITECTURE.md's pins update in the same
change". Pins updating is weaker than the document being re-shaped: after a real redesign the old
document's structure itself lies, even with fresh pins.

**Why now.** tlvphotos hit the case on 2026-07-11: a second finger bug (a moving finger drags the
underlying room layer into view in the polaroid side room; the first, resting-finger fix opened the
window for it) led Alexander to order a rethink of the UI layer architecture plus a spec update. He
asked whether the pack already forces the document rework in that situation; the honest answer was
"only partly", and he said: if it is not written, drop the wish in the inbox.

**Who throws it.** The tlvphotos Mac window (assigned to tlvphotos), 2026-07-11 ~23:11, on
Alexander's explicit word.
