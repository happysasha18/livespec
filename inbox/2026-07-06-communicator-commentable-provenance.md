# Wish (from promoter session, 2026-07-06) — communicator: work shown COMMENTABLE, with per-claim provenance

Learned on the promoter/track-coach обкатка, drew a sharp owner reaction ("открой мне всегда с
комментами и я там выскажусь… я не знаю откуда ты всё это взял"). Captured durably in
promoter-base rule 10, but it is a GENERAL communicator principle and belongs in the pack so every
project gets it (and it survives a memory wipe).

Proposed communicator rule (elaborating the existing "show, don't describe" / decision-page rules):
- Anything the human reviews (a doc, a plan, a set of drafts, a design) is shown as a surface they
  can react to LINE BY LINE — a per-item comment field + a keep/fix/cut verdict + a Save-JSON the
  session reads back — not a read-only rendered essay they can only nod at.
- Every claim carries a visible SOURCE tag, honestly typed: from the artifact (code / real output /
  shipped copy), the human's own past words, or **the assistant's own inference** — inferences
  flagged loudest, because that is where the human most needs to push back.
- Rationale: a wall of prose with no provenance reads as invented; the human can neither trust nor
  correct it. This is the round-trip (rule 10) generalized from "several open decisions" to "any
  substantial artifact shown for review."

For the live-spec window to spec + fold into communicator (and consider whether it supersedes or
merges with the existing decision-page rule 10).
